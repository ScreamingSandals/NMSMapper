NMS mapping browser

Welcome on the NMS mapping browser page. This page allows you to browse server-side mappings for Minecraft: Java Edition, and compare different mapping types and versions.

This page mainly focuses on Spigot, its forks and Spigot-Forge hybrids like MohistMC. That means, that this page doesn't contain information about things excluded in Spigot mappings (e.g. package-info). This page also doesn't contain any information about lambdas, anonymous classes or synthetic members (they are generated by the compiler, they are not in the source code).

On the overview page of every version, you can find the licenses for all the mappings and links to their origin.

If the class or the package name in the history contains ${V}, it means that there should be a CraftBukkit version string (e.g. v1_15_R1, v1_16_R3, ...). If you need the string, please open the version specific page of the class.

These mappings are provided "AS-IS", with no warranty, so mistakes are possible. We are only solving issues in classes, that we are actively using in ScreamingSandals plugins. If you want to fix anything, feel free to open a pull request or contact us on our Discord server.

Website deprecation

In June 2021 with Minecraft 1.17, Spigot's team decided to ditch their member mappings in favor of a Mojang-mapped development environment and obfuscated runtime. This change was unexpected (at least for us), and because we needed to access NMS in our plugins, we've made this project, which allowed us to browse mappings, track the history of individual symbols and automatically generate reflective accessors for multiple versions at once. However, this project has many problems, the biggest one being its memory usage and poor performance, which is getting worse with every single added version.

Therefore, one of us decided to make a new project called  takenaka. It's a successor to NMSMapper with many new features, such as support to view generics or parameter names, but mainly, it has much better performance than NMSMapper will ever have, so we have decided to deprecate this project in favor of takenaka.

Mapping browser
If you used NMSMapper to browse mappings, you can easily switch to the new mapping browser generated by takenaka. The new mapping browser is currently hosted on mappings.cephx.dev and is pretty similar to this one. This browser will be updated for patch versions of 1.20, but then we're going to wave it goodbye completely.

Reflective accessor generator
If you used NMSMapper to generate reflective accessors, you will have to adjust your build scripts and source code for takenaka. We understand this can take a lot of time, especially for larger build scripts, so we'll continue updating NMSMapper's generator for as long as you'll need it. Just let us know that you haven't migrated yet, either on our Discord server or via GitHub.

Migrating from Spigot to Mojang mappings?

Mojang <-> Spigot comparison pages are now available. On these pages, you can see all Spigot-mapped symbols and their Mojang equivalents.

These symbols are compared across multiple versions:

  • 1.16.5 - the last Spigot version with Spigot-mapped fields
  • 1.17.1 - the last Spigot version with Spigot-mapped methods
  • 1.18.2 - the latest Spigot version with Spigot-mapped class names (they may be dropped in the future)

Symbols which weren't Spigot-mapped in 1.16.5 are not compared.

Note: These pages may negatively impact your browser's performance, due to their size and processing power needed to render them.
